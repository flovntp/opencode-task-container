## Root Cause

**This is a deliberate test exception — not a code bug.**

The `TrafficSimulationController` is a load-testing endpoint designed to exercise the Auto-RCA pipeline. Its `overload()` action:

1. Validates a shared secret token (not behind admin firewall, so external traffic generators can reach it)
2. Atomically increments an APCu in-flight counter
3. Runs a CPU-intensive busy loop for ~20 seconds
4. Periodically checks the in-flight count against a configurable threshold (`TRAFFIC_SIM_MAX_CONCURRENCY`, default `1`)
5. **Throws a `RuntimeException`** when concurrency exceeds the threshold — shedding load by failing fast

The exception at 2026-06-10T09:55:07 UTC was triggered because **2 concurrent requests** hit the `/fr/admin/auto-rca/simulate-traffic-exception` endpoint while `TRAFFIC_SIM_MAX_CONCURRENCY` was `1`. The `AutoRcaSubscriber` caught the 500 error and spawned this RCA task container.

## Proposed Follow-up

This is working as designed. No code fix is required. The endpoint successfully:
- Exercised the Auto-RCA pipeline (exception → subscriber → task container)
- Demonstrated load-shedding under concurrency

If the goal is to reduce these noise triggers, consider increasing `TRAFFIC_SIM_MAX_CONCURRENCY` to match the expected PHP-FPM worker count, or disabling the simulation endpoint in production via the `TRAFFIC_SIM_TOKEN` variable.
